# Chapter 5: Pipes, Redirection & Streams

In Chapter 1, we mentioned the Linux Philosophy: **"Small tools, big impact."** This is made possible by **Streams**. In Linux, programs don't just run in isolation; they are like plumbing components that can be connected to move data from one place to another.

Understanding how to redirect these streams is what separates a basic user from a power user.

---

## 5.1 The Three Standard Streams
Every time you run a command in the terminal, Linux opens three "data pipes" for that process:

1.  **Standard Input (stdin / 0):** Data sent *into* the command (usually from your keyboard).
2.  **Standard Output (stdout / 1):** The "normal" output of the command (usually displayed on your screen).
3.  **Standard Error (stderr / 2):** Error messages generated by the command (also displayed on your screen, but through a different pipe).



---

## 5.2 Redirection: Sending Output to Files
Sometimes you don't want the output on your screen; you want it in a file.

* **`>` (Overwrite):** Redirects stdout to a file, deleting anything already in that file.
    * `ls > files.txt`
* **`>>` (Append):** Redirects stdout to a file, adding the new data to the end.
    * `echo "New entry" >> logs.txt`
* **`2>` (Error Redirection):** Redirects only the error messages to a file.
    * `ls /root 2> errors.txt` (Useful for logging what went wrong).
* **`&>` (Redirect All):** Sends both stdout and stderr to the same file.

---

## 5.3 The Pipe (`|`): Connecting Commands
The pipe operator is the most powerful tool in the Linux arsenal. It takes the **stdout** of the command on the left and feeds it directly into the **stdin** of the command on the right.

**Syntax:** `command1 | command2`

**Example:**
Suppose you want to find a specific user in a long list of logged-in users:
`who | grep "alice"`

1. `who` lists all users.
2. The `|` catches that list.
3. `grep` searches that list for "alice".

---

## 5.4 Advanced Combinations
By chaining multiple pipes, you can build a mini-program on a single line.

* **Sorting and Uniqueness:**
    `cat names.txt | sort | uniq`
    (Reads names, sorts them alphabetically, and removes duplicates).

* **Counting Results:**
    `ls /usr/bin | wc -l`
    (Lists all programs in the bin folder and counts the number of lines, telling you how many programs are installed).

* **The `tee` Command:**
    Sometimes you want to save the output to a file *and* see it on your screen at the same time.
    `ls | tee file_list.txt`

---

## 5.5 Input Redirection (`<`)
While less common than output redirection, you can also force a command to read from a file instead of the keyboard.

* `sort < unsorted_list.txt`

---

## 🛠 Hands-on: Building a Data Pipeline
Try this complex chain to see the power of pipes:

1.  **Create a messy list:**
    `echo -e "apple\norange\napple\nbanana\norange" > fruit.txt`
2.  **Process the list:**
    `cat fruit.txt | sort | uniq -c > counts.txt`
    * `sort` puts them in order.
    * `uniq -c` removes duplicates and **counts** how many times each appeared.
    * `>` saves the final result.
3.  **Check the result:** `cat counts.txt`
